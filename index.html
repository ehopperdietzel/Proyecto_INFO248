    Introducción

    Introducción a la API

    El sitio web cuenta con una API REST, para facilitar una futura implementación en dispositivos móviles.
    Se utiliza principalmente el método POST, estructurando la información de envío y respuesta en formato JSON.

    En los llamados con método POST, puede existir más de una función para un mismo servicio, los cuales se diferencian por llave "request" del mensaje JSON.

    Se indicará la nececidad de autentificación ( admin ), para algunos llamados.

    Las rutas a la cual se hacen todos los llamados se ecuentran en el directorio /api/v1/, por lo tanto todos los servicios descritos a continuación partirán desde ese directorio.

    Requisitos Funcionales

    Requisitos no Funcionales

    Iniciar Sesión

    Genera una PHP SESSION para manejar la sesión del administrador.
    El servidor retorna una cookie "admin_token", con el token de la sesión.
    Esta cookie se debe enviar en cada request para mantener la sesión.

    Método

    Método POST
    Clasificación Utilitario
    Path /admin/login
    Autentificación No

    Envío

    Key Tipo Descripción
    username String Nombre de usuario del administrador.
    password String Constraseña del administrador.

    Respuesta

    Cookie de nombre "admin_token", con el token de la sesión.

    Cerrar Sesión

    Elimina la sesión PHP del administrador, junto con sus cookies.

    Método

    Método GET
    Clasificación Utilitario
    Path /admin/logout
    Autentificación No

    Respuesta

    Retorna "Ok".

    Crear DB

    Automatiza la creación de la base de datos MySQL, junto con sus tablas y parámetros por defecto, utilizando las credenciales definidas en el archivo conf.json, ubicado en la raíz del proyecto.
    Además le asigna el formato utf8_general_ci.

    Método

    Método GET
    Clasificación Utilitario
    Path /setup
    Autentificación Administrador

    Envío

    Lee las credenciales MySQL desde el archivo conf.json, ubicado en la raíz del proyecto.

    Respuesta

    Muestra en pantalla el estado de creación de la base de datos y cada una de sus tablas.

    Listar Visibles

    Lista los productos visibles por los clientes.

    Método

    Método POST
    Clasificación Utilitario
    Path /products
    Autentificación No

    Envío

    Key Tipo Descripción
    request String Identifica la petición, debe contener "list".

    Respuesta

    Retorna un arreglo de objetos en formato JSON con los siguientes parámetros:

    Key Tipo Descripción
    id Entero ID del producto
    title String Nombre del producto
    description String Descripción del producto
    defaultPrice Entero Precio por defecto
    defaultImage String URL de la imagen por defecto
    categories Arreglo de Objetos Lista de categorías a las cuales pertenece el producto
    customizable Bool Indica si el producto es customizable
    customizations Arreglo de Objetos Lista de customizaciones junto con sus opciones
    variations Arreglo de Objetos Lista de variaciones del producto

    Donde los objetos del parámetro "categories" tiene la siguiente estructura:

    Key Tipo Descripción
    id Entero ID de la categoría
    name String Nombre de la categoría

    Los objetos del parámetro "customizations" tiene la siguiente estructura:

    Key Tipo Descripción
    id Entero ID de la customización
    name String Nombre de la customización
    options Arreglo de Objetos Lista de opciones ( id y nombre de la opción)

    Los objetos del parámetro "variations" tiene la siguiente estructura:

    Key Tipo Descripción
    id Entero ID de la variación
    productId Entero ID del producto asociado
    image String URL de la imagen alternativa
    price Entero Precio de la variación
    optionsCombination Arreglo de Objetos Lista de las combinaciónes de customizaciones que generan esta variación (ID de la costomización, ID de su opción)

    Modificar

    Modifica un slide.

    Método

    Método POST
    Clasificación Utilitario
    Path /slides
    Autentificación Administrador

    Envío

    Key Tipo Descripción
    request String Identifica la petición, debe contener "update"
    id Entero ID del slide a eliminar
    image Entero ID de la nueva imagen del slide (Opcional)
    contain Bool Modo en que se visualiza la imagen en su contenedor (Opcional)
    Si es True: La imagen se muestra completa.
    Si es False: La imagen ocupa todo el espacio del contenedor.
    visible Bool Indica si el slide será visible por los clientes (Opcional)
    position Entero Modifica el orden del slide (Opcional)

    Respuesta

    Si se elimina correctamente, retorna "Ok".
